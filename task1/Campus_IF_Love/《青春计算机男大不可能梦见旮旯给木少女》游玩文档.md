### 1. 动态剧情分支

- **起始选择**：游戏开始时，你可以选择通过“前置剧情”了解背景，或直接跳过进入角色选择。
- **路线锁定**：通过在不同场景（社团/图书馆/咖啡店）做出积极回应（选择选项1），即可锁定该角色的专属剧情线。

### 2. 互动系统

一旦进入角色路线，你可以通过以下方式提升好感度（Affinity）：

- **聊天 (Talk)**：从剧本库中随机抽取对话。选择投其所好的选项（A）可获得 **+10** 好感度，反之则会扣除。
- **送礼 (Gift)**：针对不同角色赠送特定礼物。例如：
  - **小白**最喜欢**可爱玩偶**。
  - **学姐**和**姐姐**对**精致的钢笔**更有好感。
  - *注意：赠送“奇怪的石头”会导致全员好感大跌！*

### 3. 存档与读档

- 游戏内置了 `save.json` 存档机制。
- 你可以随时保存当前的攻略进度（包括当前锁定的角色和所有人的好感度数值），并在下次启动游戏时无缝衔接。
- 也可以删除当前的存档

### 4. 结局系统

- 当某位角色的好感度达到 **100** 时，主菜单将解锁 **“0. 迈入结局”** 选项。
- 每位角色都有独特的结局文本，描绘你们未来的共同生活。

这个功能的核心在于如何把玩家这时候游玩的状态记录下来，这个游戏最重要的就是这几个数据

如果只是单纯的想要存储下这些数据其实很简单，在外部直接开一个txt文件都能随便存，可问题是，这样子的话还叫存档吗，存是存了，那档呢？

因此我们这个时候需要一种可以与.py程序进行交互的文件类型，支持序列化与反序列化，存档的时候我们把关键信息写进去，读档的时候就把它里面包含的信息交给.py文件，也就是反序列化。

json就是一个很好的类型，而如果要用到json，那么就需要在.py文件中导入json库：

```python
import json
```

哈基米告诉我的方法有这些

| **方法名**         | **动作 (Action)**   | **输入/输出对象**                       | **形象比喻**                   |
| ------------------ | ------------------- | --------------------------------------- | ------------------------------ |
| **`json.dumps()`** | **序列化** (编码)   | 将 Python 对象转为 **JSON 字符串**      | 把礼物装进快递盒（字符串）     |
| **`json.dump()`**  | **序列化** (写入)   | 将 Python 对象转为 **JSON 文件**        | 把礼物装进快递盒并**邮寄出去** |
| **`json.loads()`** | **反序列化** (解码) | 将 **JSON 字符串** 转为 Python 对象     | 拆开快递盒（字符串）拿礼物     |
| **`json.load()`**  | **反序列化** (读取) | 从 **JSON 文件** 读取并转为 Python 对象 | 从**邮寄来的**快递盒里拿礼物   |

存档的操作非常简单，就是利用dump方法把数据写入到json文件中去就行，由于这是个单线的游戏，因此（可能？）只要写入对应角色线角色的好感度就行了。但是为了后续能够实现一些优化（例如加什么共通线，虽然目前是单线游戏（再次强调）），还是建议把所有角色的好感度都存起来比较好。

代码实现是这样的

```python
def save_game(self, filename="save.json"):
        """保存当前角色状态和所有人的好感度"""
        data={
            "current_target": self.current_target.name,
            "affinities": {name: char.affinity for name, char in self.characters.items()}
        }#字典推导式，同时获取Game类里的角色列表的键名和Character类里面的好感度选项
        with open(filename,"w",encoding="utf-8") as f:#with表达式可以实现的是我们对json文件操作完成之后安全关闭
            json.dump(data,f,ensure_ascii=False)#第三个参数保证的是中文可见
        print("存档成功")
```

这样子就实现了存档，读档的逻辑其实就是去读取json文件中现在处于的角色线，以及每个角色的好感度，同时把这个好感度替换到py主程序里面去。
