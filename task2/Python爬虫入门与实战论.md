## 2.requests库

  requests库是爬虫程序的核心，少了谁都不能少了它，使用它需要我们先在虚拟环境里面去下载安装

  我这里用的conda管理的虚拟环境，只要进入对应的虚拟环境编辑界面，然后输入

```
conda install requests
```

  就可以了，然后只要在vscode的页面配置一下解释器和环境就可以了。

  在爬虫的主程序里面导入requests模块

```python
import requests
```

  接下来需要获取url，这里以福大教务处：https://jwch.fzu.edu.cn/做例子

```python
import requests
import os
if __name__=="__main__":
    url="https://jwch.fzu.edu.cn/"
    response=requests.get(url=url)
    response.encoding='utf-8'
    page=response.text
    filename=os.path.join("save","fzu.html")
    with open(filename,"w",encoding="utf-8") as f:
        f.write(page)
```

  requests库的get函数里面有三个参数，不过只填了一个也可以就是，第一个参数url，其实就是你要让函数解析获取的页面。

  第六行其实不是必要的，为什么加了这个呢，因为你福州大学教务处是utf-8和gbk混用的啊~不加这个会爬下来一堆乱码的喵。

  后面的逻辑就不讲了

  我们说get后面有三个参数，那么第二个和第三个是什么？第二个参数可以帮助我们跳转到某一个指定的子页面，需要以一个字典形式传入，同时我们还要去知道这个网址后面的子页面是什么样的格式。

  第三个参数是干什么用的，用来干UA伪装的，这个也是要用字典的形式传进去。让页面知道你不是一个爬虫程序，是一个人正在通过浏览器访问指定的网址，不然有些时候会被反爬机制直接干掉（备注：教务处没有反爬）

## 3.什么是阿贾克斯请求

  AJAX的核心是异步刷新，可以用这样子几个方法去判断一个请求是否是一个AJAX请求。

  判断一个请求是不是 **AJAX（阿贾克斯）**，最核心的思路只有两个：**看网页行为**（外行看热闹）和**看开发者工具**（内行看门道）。

------

- **1. 观察网页行为（最直观）**

如果网页在不刷新整个页面的情况下，内容发生了变化，那基本就是 AJAX。

- **瀑布流加载**：当你滚动鼠标到底部，新内容自动弹出来（如小红书、微博、抖音网页版）。
- **局部刷新**：点击“加载更多”按钮，或者在搜索框打字时下拉框自动弹出候选词，而浏览器顶部的刷新图标没动。
- **点击不跳转**：点击了某个按钮（比如点赞、收藏），页面没跳到新网址，但图标变色了。

------

- **2. 使用 Chrome 开发者工具（最准确）**

这是爬虫工程师最常用的方法，直接看数据包的“户口本”。

1. 按 `F12` 打开开发者工具。

2. 点击顶部的 **Network（网络）** 面板。

3. 点击下方的 **Fetch/XHR** 过滤按钮。

   > **科普**：AJAX 的核心技术叫 `XMLHttpRequest` (XHR)，现代浏览器多用 `Fetch API`。点击这个按钮后，显示的列表**全是 AJAX 请求**。

4. 刷新页面或执行触发动作（如点翻页），观察列表里跳出的新请求。

------

- **3. 查看请求头（进阶判断）**

有些老旧或传统的 AJAX 请求会在 HTTP Header 中贴上一个“特殊标签”：

在 **Headers** 选项卡下找到 **Request Headers**，寻找这一行：

```
X-Requested-With: XMLHttpRequest
```

- 如果有这一行，说明这是一个典型的 AJAX 请求。
- **注意**：现代的 `fetch` 请求可能不再携带这个字段，所以主要还是靠 `Fetch/XHR` 过滤来判断。

------

- **4. 终极验证法：查看响应内容 (Preview)**

点击那个可疑的请求，切换到 **Preview（预览）** 或 **Response（响应）** 选项卡：

- **普通请求**：返回的是一整套 HTML 代码（包含 `<html>`, `<body>` 等）。
- **AJAX 请求**：通常返回的是纯数据，最常见的是 **JSON 格式**（像字典一样的一对对数据），或者是纯文本、一小段 HTML 碎片。



AJAX是好东西啊，这个东西可以直接让我们获取到数据包而不用对着一大堆奇怪的HTML思考人生。

## 4.GET与POST请求

| **维度**     | **GET 请求**                               | **POST 请求**                                |
| ------------ | ------------------------------------------ | -------------------------------------------- |
| **主要功能** | **获取**数据（如：搜索、翻页、读文章）     | **提交**数据（如：登录、发帖、上传文件）     |
| **参数位置** | **URL 后面**（用 `?` 连接，`&` 分隔）      | **请求体 (Body)** 中（地址栏看不见）         |
| **数据长度** | 有限制（通常 URL 限制在 **2KB** 左右）     | 理论上**无限制**（适合发长文、大图）         |
| **安全性**   | **低**（参数直接暴露，且留在浏览器历史里） | **相对高**（数据不刷脸，但仍需 HTTPS 加密）  |
| **幂等性***  | **幂等**（请求 1 次和 100 次结果一样）     | **非幂等**（请求 2 次可能发了 2 篇重复帖子） |
| **缓存**     | **会被缓存**（下次访问会快很多）           | **不会缓存**                                 |

  简单来说就是，GET请求是你直接输入网址，然后就给你信息，而POST请求的话，就要先索取信息，通常需要在`requests.post()`这个函数内部就传入它需要的信息，它才能给你要的信息，因此在`requests`里面，这两个函数内部传的参数也有差别

  get里面传入的是url和params关键字，而post里面要先传入url，然后根据需要的数据类型传入data（要以字典的形式传进去）或者是json，然后这俩函数最后都可以放一个header关键字用来做ua伪装。

## 5.XPATH

| **表达式** | **描述**     | **举例**         | **说明**                                              |
| ---------- | ------------ | ---------------- | ----------------------------------------------------- |
| `/`        | **绝对路径** | `/html/body/div` | 从根节点开始，一级一级往下找，比较死板。              |
| `//`       | **相对路径** | `//li`           | 在全局范围内找所有的 `li`，不管它在哪。**（最常用）** |
| `.`        | **当前节点** | `./a`            | 从当前已经选中的标签开始往下找。                      |
| `..`       | **父节点**   | `./..`           | 回到当前标签的上一级。                                |
| `@`        | **选取属性** | `//@href`        | 提取标签里的属性值（如链接、图片地址等）。            |

| **表达式**      | **描述**     | **举例**                                            |
| --------------- | ------------ | --------------------------------------------------- |
| `[n]`           | **按序号**   | `//li[1]` （注意：XPath 序号从 **1** 开始，不是 0） |
| `[last()]`      | **最后一个** | `//li[last()]` 选取列表里的最后一项。               |
| `[@attr]`       | **按属性名** | `//a[@title]` 选取所有带 title 属性的链接。         |
| `[@attr="val"]` | **按属性值** | `//div[@class="content"]` 选取特定的 class 盒子。   |

| **函数**        | **描述**         | **举例**                                                     |
| --------------- | ---------------- | ------------------------------------------------------------ |
| `contains()`    | **包含**匹配     | `//div[contains(@class, "box")]` 匹配 class 包含 "box" 的 div。 |
| `text()`        | **获取文本**     | `//a/text()` 拿走链接中间显示的文字。                        |
| `//text()`      | **提取所有文本** | `//div//text()` 拿走 div 及其所有子标签里的文字。            |
| `starts-with()` | **开头**匹配     | `//input[starts-with(@id, "user_")]`                         |

| **表达式**            | **描述**     | **举例**                                                    |
| --------------------- | ------------ | ----------------------------------------------------------- |
| `following-sibling::` | **后续兄弟** | `//span/following-sibling::a` 找 span 后面的第一个 a 标签。 |
| `preceding-sibling::` | **前置兄弟** | `//a/preceding-sibling::span` 找 a 标签前面的那个 span。    |
